# Спецификация стандарта ACS-1

**Название:** Alternet Color Standard (ACS-1)  
**Статус:** Черновик / Примитивный профиль  
**Цель:** Единое управление базовыми цветами документа в среде CommonMark.

## 1. Общие положения
Стандарт ACS-1 позволяет автору документа задать три базовых параметра отображения: цвет фона, цвет текста и цвет ссылок. Информация о стилях хранится внутри файла CommonMark в виде специального HTML-комментария в первой строке документа.

Почему комментарий?
1.  Это валидный синтаксис CommonMark (игнорируется при рендеринге текста).
2.  Это легко извлечь программно (regex).
3.  Это не требует введения новых символов экранирования.

## 2. Синтаксис
Стилевая директива должна находиться **в самой первой строке** файла.

Формат строки:
```markdown
<!-- ACS: bg=#XXXXXX; text=#XXXXXX; link=#XXXXXX -->
```

Где:
*   `<!-- ACS:` — обязательная сигнатура начала.
*   `bg` — цвет фона (background).
*   `text` — цвет основного текста.
*   `link` — цвет гиперссылок.
*   `#XXXXXX` — код цвета в формате HEX (6 символов, регистр не важен).
*   `;` — разделитель параметров.
*   `-->` — обязательное окончание комментария.

**Правила:**
1.  Пробелы вокруг знаков `=` и `;` допускаются, но не обязательны.
2.  Порядок параметров не важен.
3.  Если параметр отсутствует, используется значение по умолчанию (браузерное или системное).
4.  Все последующие строки файла считаются обычным контентом CommonMark.

## 3. Параметры

| Параметр | Описание | Пример значения |
| :--- | :--- | :--- |
| `bg` | Цвет фона страницы | `#FFFFFF`, `#000000` |
| `text` | Цвет основного текста | `#000000`, `#CCCCCC` |
| `link` | Цвет активных ссылок | `#0000FF`, `#00FF00` |

## 4. Руководство по реализации (для разработчиков браузеров/рендереров)

Поскольку браузеры не рендерят CommonMark нативно, реализация ложится на **просмотрщик Alternet** (расширение браузера, шлюз или приложение).

**Алгоритм обработки:**
1.  При загрузке документа прочитать первую строку.
2.  Проверить наличие сигнатуры `<!-- ACS:`.
3.  Если сигнатура найдена:
    *   Извлечь значения параметров через регулярное выражение.
    *   Сгенерировать блок CSS стилей.
    *   Внедрить стили в `<head>` результирующей HTML-страницы.
4.  Удалить первую строку из контента перед конвертацией CommonMark в HTML (чтобы комментарий не отображался пользователю).

**Пример генерируемого CSS:**
```css
body {
    background-color: [значение bg];
    color: [значение text];
}
a {
    color: [значение link];
}
```

## 5. Пример документа

**Исходный код (файл .md):**
```markdown
<!-- ACS: bg=#1a1a1a; text=#e0e0e0; link=#4da6ff -->

# Приветствие

Это документ в сети **Alternet**.

Здесь используется темная тема, заданная стандартом ACS-1. 
Перейдите на [главную страницу](#), чтобы увидеть эффект ссылки.
```

**Результат (визуально):**
*   Фон: Темно-серый (`#1a1a1a`).
*   Текст: Светло-серый (`#e0e0e0`).
*   Ссылка: Голубая (`#4da6ff`).
*   Заголовок и жирный текст рендерятся стандартными средствами CommonMark.

## 6. Преимущества стандарта ACS-1

1.  **Человекопонятность:** Любой пользователь может открыть файл в текстовом редакторе и сразу увидеть настройки цвета.
2.  **Легковесность:** Не требует подключения внешних файлов стилей.
3.  **Совместимость:** Файл остается валидным CommonMark даже для тех систем, которые не поддерживают ACS (они просто проигнорируют комментарий).
4.  **Простота кода:** Для поддержки стандарта достаточно 10-15 строк кода на JavaScript или любом другом языке.
